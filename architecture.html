Structured Data Extraction Pipeline — Architecture

CCA-F Exercise 3 · reliable extraction, not a perfect model · claude-sonnet-4-6 + pydantic

tool_use + nullable → no fabrication

A schema-bound tool_use guarantees valid JSON (no parse errors). Fields that may be absent are Optional/nullable with "if absent, null" — so the model returns null instead of inventing a value.

resolvable vs unresolvable errors

Format mismatch (wrong type/enum) → retry with the specific error fixes it. Info absent from source → retry never fixes it; retrying is an infinite-loop money trap. Classify, then route the unresolvable to a human.

field-level confidence & accuracy

Per-field confidence routes low-confidence extractions to human review. Measure accuracy by doc_type × field, not a single aggregate — 95% overall can mask a 40% cell.
